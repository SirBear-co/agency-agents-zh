# Agent Picking Guide

遇到一个新 idea 时，先不要纠结“最完美的 agent 是谁”，而是先找最接近主问题的那个。

## 常见问题怎么选

### 内容账号 / 社交媒体

- `marketing-content-creator`：账号定位、内容方向、栏目设计
- `marketing-xiaohongshu-operator`：小红书平台打法、选题、封面、起号策略
- `design-brand-guardian`：账号气质、命名、品牌感

### 产品 / 服务 / 商业模式

- `product-manager`：产品定义、目标用户、路线图
- `product-feedback-synthesizer`：用户反馈归类与洞察
- `sales-deal-strategist`：商业化逻辑、成交场景

### 用户理解 / 人群画像

- `design-ux-researcher`：用户访谈框架、行为洞察
- `academic-psychologist`：动机、情绪、行为模式

### 写作 / 叙事 / 表达

- `marketing-content-creator`：内容输出
- `marketing-book-co-author`：长篇表达、章节结构
- `academic-narratologist`：叙事结构、故事张力

### 品牌 / 视觉 / 风格

- `design-brand-guardian`：品牌定位
- `design-ui-designer`：视觉方向
- `design-visual-storyteller`：信息表达与视觉叙事

### 工具 / 网站 / App / 技术实现

- `engineering-rapid-prototyper`：快速验证
- `engineering-frontend-developer`：前端界面
- `engineering-backend-architect`：后端架构
- `engineering-software-architect`：整体系统设计

## 一个简单判断法

先问自己：这个 idea 当前最卡的是哪一类问题？

- 不知道做什么内容：先找 `marketing-content-creator`
- 知道做什么，但不知道平台怎么玩：找对应平台 agent
- 想法很多，但用户是谁不清楚：找 `design-ux-researcher`
- 想落成产品：找 `product-manager`
- 想开始做代码：找工程类 agent

## 推荐顺序

大多数 idea 都可以按这个顺序推进：

1. `marketing-content-creator` 或 `product-manager` 先帮你把想法说清
2. 找垂直 agent 细化执行方案
3. 找品牌/设计 agent 做外在表达
4. 真要落地时再找工程 agent
